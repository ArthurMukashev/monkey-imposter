# Проект: Бэкенд + CMS для афиши (туризм, активный отдых, гастрономия)

## Краткое описание

Этот проект представляет собой бэкенд-систему и CMS для управления контентом афиши. Он предназначен для публикации объектов по трём направлениям: туризм, активный отдых и гастрономия. Система включает админ-панель на **Filament** и публичное **REST API** с документацией.

---

## Технологический стек

- **PHP 8.4**
- **Laravel 12** – основной фреймворк
- **Filament** – админ-панель (CMS)
- **PostgreSQL 16 + PostGIS** – база данных с поддержкой пространственных данных
- **Nginx** – веб-сервер
- **Docker & Docker Compose** – контейнеризация
- **Scramble** – автоматическая генерация OpenAPI-документации

---

## Используемые библиотеки (основные)

| Пакет | Назначение |
|-------|------------|
| `filament/filament` | Административная панель |
| `laravel/scramble` | Документация API (OpenAPI) |

## Документация API

После запуска проекта документация доступна по адресу:

```
http://localhost/docs/api
```

Она генерируется автоматически с помощью **Scramble** и содержит все эндпоинты, параметры, примеры ответов.

## Инструкция для фронтендера

### 1. Клонирование репозитория

```bash
git clone <url-репозитория>
cd <папка-проекта>
```

### 2. Настройка окружения

Скопируйте `.env.example` в `.env`:

```bash
cp .env.example .env
```

При необходимости отредактируйте параметры (база данных уже настроена для Docker).

### 3. Запуск Docker-контейнеров

```bash
docker-compose up -d --build
```

После запуска будут доступны:

- Приложение: [http://localhost](http://localhost)
- Админ-панель: [http://localhost/admin](http://localhost/admin)
- PgAdmin: [http://localhost:8080](http://localhost:8080) (email: `admin@example.com`, пароль: `admin`)
- Документация API: [http://localhost/docs/api](http://localhost/docs/api)

### 4. Создание пользователя для Filament (администратора)

В контейнере выполните скрипт:

```bash
docker-compose exec app ./create-filament-user.sh
```

Скрипт создаст пользователя с данными:

- **Email:** `admin@mail.ru`
- **Пароль:** `admin`

Теперь вы можете войти в админ-панель по адресу `/admin`.

### 5. Запуск миграций (если не выполнены автоматически)

```bash
docker-compose exec app php artisan migrate
```

### 6. Дополнительные команды

- Остановка контейнеров: `docker-compose down`
- Просмотр логов: `docker-compose logs -f`
- Перезапуск после изменений: `docker-compose restart`

## Структура API

Все публичные эндпоинты доступны по префиксу `/api/v1`. Основные маршруты:

| Метод | Эндпоинт | Описание |
|-------|----------|----------|
| GET | `/places` | Список объектов (с фильтрацией и пагинацией) |
| GET | `/places/{slug}` | Детальная карточка объекта |
| GET | `/cities` | Справочник городов |
| GET | `/categories` | Справочник категорий |
| GET | `/tags` | Справочник тегов |
| GET | `/cuisine-types` | Справочник типов кухонь |
| GET | `/promos` | Промо-блоки (по месту показа) |
| GET | `/health` | Проверка доступности |

Подробное описание параметров и форматов ответов смотрите в документации `/docs/api`.
